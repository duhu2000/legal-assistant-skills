# 法律AI求职助手 (legal-job-search)

> 帮助法律人（法务/律师）使用AI辅助求职的 Claude Code 技能

---

## 功能概述

本技能提供完整的AI辅助求职流程，支持：

| 功能 | 说明 |
|------|------|
| **公司/律所调研** | 支持 Deep Research 降级到 Web Search |
| **法律风险分析** | 支持 MCP 工具级降级到 Web Search |
| **网页简历生成** | 完全可配置的样式系统 |
| **针对性简历调整** | 基于 JD 和目标公司/律所定制 |
| **面试备忘录生成** | 结构化面试准备材料 |

---

## 快速开始

### 场景 1：法务岗位求职

```
用户：我想申请腾讯音乐的法务岗位，JD 在这里

Claude 执行流程：
1. 提取 JD 信息
2. 询问：你有秘塔AI搜索或 Perplexity Pro 吗？
3. 公司调研：
   - 有 Deep Research：使用秘塔AI/Perplexity
   - 无 Deep Research：使用 Web Search（多轮）
4. 法律风险分析：
   - 尝试每个 MCP 工具
   - 失败的 MCP 单独降级到 Web Search
5. 询问：你的基础简历信息
6. 生成：
   - 网页简历（选择样式）
   - 面试备忘录
   - 针对性简历调整
```

### 场景 2：律所岗位求职

```
用户：我想申请红杉律所的资本市场团队

Claude 执行流程：
1. 提取 JD 信息（如有）
2. 使用「律所调研框架」收集公开信息
3. 提供「团队信息收集清单」引导用户补充私域信息
4. 生成面试备忘录和针对性简历
```

### 场景 3：快速调研模式

```
用户：帮我快速调研一下字节跳动的法律风险情况

Claude 执行：
1. 直接执行法律风险分析（工具级降级）
2. 输出法律风险分析报告
```

## MCP 工具支持

### 核心工具（企业法律风险类）

| MCP 工具 | 功能 | 降级查询 |
|---------|------|----------|
| `get_executed_entity` | 企业被执行信息 | `"{公司名}" 被执行人信息` |
| `get_dishonest_entity` | 企业失信信息 | `"{公司名}" 失信被执行人名单` |
| `get_consumption_restrictions` | 限制高消费 | `"{公司名}" 限制高消费令` |
| `get_final_cases` | 终本案件 | `"{公司名}" 终本案件` |
| `get_judgments` | 裁判文书 | `"{公司名}" 裁判文书` |

### 工具级降级策略

```
for 每个 MCP 工具:
    │
    ├─ 尝试调用 MCP
    │   ├─ 成功 → 记录结果
    │   └─ 失败/不存在 → 执行降级
    │
    └─ 降级策略:
        └─ Web Search (针对性查询)
        └─ 整合信息到报告
```

## 简历样式系统

### 预设样式

| 样式名称 | 风格特点 | 适用场景 |
|---------|---------|----------|
| **business-elite** | 商务精英风（金色+深蓝） | 法务、金融、传统行业 |
| **modern-tech** | 现代科技风（蓝色+灰白） | 科技公司、互联网企业 |
| **minimal-clean** | 极简清新风（绿色+米白） | 通用、初创企业 |

### 自定义样式

完全可配置的样式系统，支持自定义：
- 配色方案（主色、辅色、中性色）
- 字体方案（字体族、字号、字重）
- 布局参数（页面、头部、侧边栏、间距）
- 渐变与特效（阴影、边框）

详见：`references/styles/custom-guide.md`

## 使用建议

1. **信息真实性**：所有内容必须基于真实信息，严禁虚构经历
2. **数据验证**：Web Search 获取的数据建议二次验证
3. **面试导向**：生成的材料要能直接支撑面试对话
4. **持续更新**：根据面试反馈调整简历和备忘录
