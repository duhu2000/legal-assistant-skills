# 律所求职扩展 - 团队信息收集

> 适用场景：律所岗位求职
> 核心问题：律所工作以团队划分，公开信息有限，需补充私域信息

---

## 团队信息收集框架

### 一、公开信息收集（AI可执行）

#### 1.1 律所层面
```
信息源：
├─ 律所官网（关于我们、团队介绍、业务领域）
├─ 律协公示信息（律所登记、律师名单）
├─ 行业媒体/榜单（钱伯斯、ALB、Legal500）
├─ 招聘平台（律所介绍、在招职位）
└─ 社交媒体（公众号、知乎、领英）
```

**收集内容**：
- 律所规模（人数、分所、业务领域）
- 行业定位/特色
- 核心客户群体
- 典型案例/业绩

#### 1.2 团队层面（如有）
```
信息源：
├─ 官网团队介绍页
├─ 律师个人简介页
├─ 专业文章/案例分析
└─ 媒体采访/演讲
```

**收集内容**：
- 团队负责人（姓名、职位、从业年限）
- 团队规模（律师人数、辅助人员）
- 业务领域（主要业务类型）
- 典型案例（公开案例）

---

### 二、私域信息收集（用户自行补充）

#### 2.1 团队负责人
```
信息收集清单：
├─ 基本信息
│   ├─ 姓名、职位、从业年限
│   ├─ 教育背景（院校、专业）
│   └─ 专业资质（律师证、其他资格）
│
├─ 专业信息
│   ├─ 主要业务领域（2-3个）
│   ├─ 典型案例/项目（3-5个）
│   ├─ 主要客户类型
│   └─ 行业声誉/媒体曝光
│
└─ 个人风格（如可知）
    ├─ 工作风格（细致/果断/亲和...）
    ├─ 带徒风格（放权/指导/严格...）
    └─ 价值观倾向
```

#### 2.2 团队业务
```
信息收集清单：
├─ 业务类型
│   ├─ 主要业务领域（诉讼/非诉比例）
│   ├─ 客户结构（行业分布、规模分布）
│   └─ 业务来源（律所分配/自拓/推荐）
│
├─ 典型案例
│   ├─ 近期主要案例（3-5个）
│   ├─ 案例类型分布
│   └─ 在案例中的角色（主办/协办）
│
└─ 收费模式
    ├─ 计费方式（计时/固定/风险代理）
    ├─ 收费水平（市场定位）
    └─ 客户付费能力/意愿
```

#### 2.3 团队文化
```
信息收集清单：
├─ 工作强度
│   ├─ 典型工作时长
│   ├─ 加班频率（工作日/周末）
│   └─ 忙闲节奏（项目周期性）
│
├─ 协作模式
│   ├─ 内部分工（主办/协办/辅助）
│   ├─ 协作方式（会议/文档/即时通讯）
│   └─ 决策机制（民主/集中）
│
├─ 培养体系
│   ├─ 带教方式（一对一/团队培养）
│   ├─ 培训内容（专业/技能/综合）
│   └─ 晋升路径（助理/律师/合伙人）
│
└─ 离职流向（如可知）
    ├─ 离职去向（其他律所/法务/转行）
    ├─ 离职原因（如可知）
    └─ 离职关系（保持联系/疏远）
```

---

### 三、信息收集方法建议

#### 3.1 正式渠道
```
├─ 招聘面试时询问
├─ 联系HR或招聘负责人
└─ 咨询律所公共关系部门
```

#### 3.2 非正式渠道
```
├─ 联系前员工/同事
├─ 询问同行/朋友
├─ 查看社交媒体（知乎脉脉领英）
└─ 参加行业活动/讲座
```

#### 3.3 观察渠道
```
├─ 观察律所办公环境（如有机会）
├─ 观察员工状态（工作时间/氛围）
└─ 观察客户评价（公开评价/口碑）
```
